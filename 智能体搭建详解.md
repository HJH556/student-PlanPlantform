# 智能体搭建详解

## 一、整体架构

项目包含三个核心智能体，底层架构统一：LangChain框架 + 通义千问大模型（qwen-turbo）+ DashScope API。

三个智能体分别是：
- **简历分析智能体（ResumeAnalyzer）**：解析简历，生成个人画像
- **职业规划顾问智能体（CareerAdvisor）**：生成职业规划、匹配度分析、技能提升建议
- **面试模拟智能体（InterviewSimulator）**：生成面试问题、评估回答、模拟对话

搭建顺序按照从底层到上层的逻辑：配置层 → LLM调用层 → Prompt设计 → Chain组装 → 输出解析 → 数据标准化 → 降级策略 → API暴露。

---

## 二、搭建过程

### 第一步：统一配置层

做一个配置类，把所有智能体共用的参数集中管理。包括DashScope的API Key、模型名称、模型参数、Prompt模板。

关键参数设置：temperature设成0.1减少随机性，max_tokens设成2000保证输出长度。base_url指向DashScope的兼容接口地址。

这样做的好处是，如果后续要换模型或者调参数，只需要改一个地方，所有智能体都会生效。

### 第二步：搭建LLM调用层

LangChain本身支持OpenAI格式的API调用，通义千问的DashScope提供了兼容OpenAI的接口，所以直接用LangChain的ChatOpenAI类，把base_url指向DashScope的地址就打通了。

简历分析和职业规划用的ChatOpenAI类，面试模拟用的Tongyi类（LangChain社区提供的通义千问原生接口），两者功能一样，只是调用方式略有不同。

### 第三步：设计Prompt模板

这是整个智能体搭建里最核心的一步。Prompt的质量直接决定了大模型输出的质量。

Prompt设计思路是：**角色定义 + 输入数据 + 输出格式要求 + 约束条件**。

**简历分析的Prompt**要求大模型返回严格JSON格式，包含6大模块：
- 基本信息（姓名、学历、专业、工作经验、技能特长）
- 能力评估（技术能力、沟通能力、团队协作、学习能力，每项0-100打分）
- 职业倾向（兴趣领域、职业目标、5个匹配岗位及匹配度）
- 匹配度分析（总体匹配度、优势分析、改进建议、技能匹配列表）
- 可视化数据（雷达图、柱状图、进度条的数据）
- 行动计划（短期/中期/长期任务列表）

Prompt里特别强调了约束条件：必须返回纯JSON格式不要包含其他文本、所有分数必须是0-100整数、所有字段必须有具体内容不要使用"待分析"。

**职业规划的Prompt**针对不同服务设计了三个模板：
- 职业规划模板：要求输出职业发展路径（短/中/长期）、技能提升建议、岗位匹配分析、风险评估
- 匹配分析模板：要求输出匹配度评分（0-100）、优势分析、改进建议、推荐岗位
- 技能提升模板：要求输出技能现状分析、学习路径规划、实践项目建议、能力提升时间线

**面试模拟的Prompt**也是三个模板：
- 问题生成模板：要求输出问题内容、类型、难度、回答建议、详细解析、评估标准
- 回答评估模板：要求输出0-100分数、优点列表、不足列表、改进建议、总体评价
- 对话模拟模板：要求输出下一个问题、追问列表、面试官反馈、对话总结

### 第四步：组装LLM Chain

Prompt模板设计好之后，用LangChain的PromptTemplate和LLMChain把它们串起来。

PromptTemplate定义输入变量和模板文本，LLMChain把PromptTemplate和LLM模型绑定。调用的时候只需要传入变量，Chain会自动把变量填到Prompt里，调用LLM，返回结果。

每个智能体根据功能不同，组装了不同的Chain：
- 简历分析：1个Chain（简历分析）
- 职业规划：3个Chain（职业规划、匹配分析、技能提升）
- 面试模拟：3个Chain（问题生成、回答评估、对话模拟）

### 第五步：输出解析

大模型返回的是纯文本，需要解析成结构化数据。

简历分析和面试模拟都要求大模型返回JSON格式，但大模型的输出是不稳定的——有时候返回标准JSON，有时候JSON前面加一句"好的，以下是分析结果："，有时候格式还不完全正确。

所以做了**三层解析容错**：

**第一层**：直接JSON解析。覆盖大模型正常返回标准JSON的情况。

**第二层**：用正则表达式从文本中提取JSON对象。找到第一个{到最后一个}之间的内容，再做JSON解析。覆盖大模型在JSON前后加了解释文字的情况。

**第三层**：正则逐字段提取。用正则匹配"## 基本信息"、"## 能力评估"等章节标题，提取各章节的文本内容，再用键值对解析。覆盖JSON格式完全损坏的情况。

面试模拟智能体还做了一个OutputParser类，继承LangChain的BaseOutputParser，把解析逻辑封装进去，Chain调用时自动解析。

职业规划智能体的输出是Markdown格式而不是JSON，所以解析方式不同——用正则匹配各章节标题，提取章节内容作为结果。

### 第六步：数据标准化

解析完之后还需要做一步标准化处理，因为大模型返回的数据格式不统一。

主要处理三类问题：

**类型不统一**：匹配度有时候是字符串"85%"，有时候是数字85，前端统一需要数字。所以在后端统一做转换，把字符串类型的分数转成整数。

**字段名不统一**：同一个字段大模型有时候叫"匹配度"，有时候叫"match_score"。标准化层做了一个互转逻辑，确保两个字段都存在。

**缺失字段**：大模型有时候会漏掉某些字段。标准化层会检查关键字段是否存在，不存在就补默认值。比如技能匹配项里如果没有"重要性"字段，就默认填"中"。

### 第七步：降级策略

每个智能体都内置了默认数据，AI调用失败时返回预设内容，确保接口不会报错。

**简历分析降级**：AI分析失败时返回一个基础的分析结果，包含默认的能力评分和职业建议。接口层还有额外的容错——检查文本长度够不够、检查是不是文件URL。

**职业规划降级**：AI调用失败时返回默认的职业规划模板，包含通用的短/中/长期目标和技能提升建议。

**面试模拟降级**：问题生成失败时返回两道通用面试题——"请介绍一下您对这个岗位的理解"和"请描述一个您解决过的技术难题"。回答评估失败时返回一个70分的默认评估结果。

这个设计确保了即使DashScope API挂了，系统仍然能正常运行，用户不会看到白屏。

### 第八步：暴露API接口

智能体开发完之后，在FastAPI路由层包装成HTTP接口。

接口路由设计：

| 路由 | 功能 | 调用智能体 |
|------|------|-----------|
| POST /langchain/analyze-resume | 简历分析 | ResumeAnalyzer |
| POST /langchain/generate-career-plan | 职业规划 | CareerAdvisor |
| POST /langchain/analyze-job-match | 匹配度分析 | CareerAdvisor |
| POST /langchain/suggest-skill-improvement | 技能提升 | CareerAdvisor |
| POST /langchain/recommend-jobs | 岗位推荐 | CareerAdvisor |
| POST /interview/generate-questions | 生成面试题 | InterviewSimulator |
| POST /interview/evaluate-answer | 评估回答 | InterviewSimulator |
| POST /interview/simulate-dialogue | 模拟对话 | InterviewSimulator |
| GET /interview/health | 健康检查 | InterviewSimulator |

路由层还做了一层额外的容错，每个接口都有try-except包裹，确保任何情况下都能返回有效响应。

## 三、知识图谱智能体

除了上面三个基于LangChain的智能体，还有一个知识图谱模块。这个模块不是传统的LLM Chain，而是结合了Neo4j图数据库和大模型的混合架构。

**第一步：构建图谱**

在Neo4j里建节点和关系。节点是岗位（Job），关系有三种：晋升（PROMOTES_TO）、转岗（TRANSFERS_TO）、相关（RELATED_TO）。覆盖了10多种IT岗位类型的多级晋升路径和转岗方向。

**第二步：意图识别**

用户提问时，先做意图判断——同时包含岗位关键词和职业发展关键词（晋升、转岗、发展、方向等）就走图谱查询，否则走大模型对话。

**第三步：Cypher查询生成**

图谱查询需要Cypher语句，有两种生成方式：

优先方式：调通义千问把自然语言转成Cypher。把用户问题和Neo4j的schema信息一起发给大模型，让它生成对应的查询语句。

备用方式：正则从问题里提取岗位名称，根据关键词判断查询类型——包含"晋升"就查PROMOTES_TO关系，包含"转岗"就查TRANSFERS_TO关系，包含"相关"就查RELATED_TO关系。

**第四步：结果格式化**

把Neo4j返回的图数据格式化成用户可读的文本，返回给前端展示。

## 四、整体数据流

以用户上传简历到完成面试模拟的完整流程为例：

1. 用户上传简历（PDF/Word/图片）
2. 前端提取文本（pdfjs-dist解析PDF、mammoth解析Word、tesseract.js做OCR）
3. 文本发到后端，调用简历分析智能体
4. Prompt填入简历文本，调用通义千问，返回JSON
5. 三层解析 + 数据标准化，生成个人画像
6. 前端拿到画像数据，展示能力雷达图、匹配岗位、行动计划
7. 用户选择目标岗位，调用职业规划智能体
8. 个人画像 + 目标岗位填入Prompt，调用通义千问，返回职业规划
9. 用户进入面试模拟，调用面试问题生成智能体
10. 简历 + 岗位信息填入Prompt，调用通义千问，返回个性化面试题
11. 用户回答问题，调用回答评估智能体
12. 问题 + 用户回答 + 参考答案填入Prompt，调用通义千问，返回评估结果

## 五、关键技术总结

| 技术点 | 实现方式 |
|-------|---------|
| LLM调用 | LangChain ChatOpenAI / Tongyi + DashScope API |
| Prompt管理 | PromptTemplate模板化，角色+输入+格式+约束 |
| 链式调用 | LLMChain组装Prompt和LLM |
| 输出解析 | 三层容错：直接解析 → 正则提取 → 字段回退 |
| 数据标准化 | 类型转换、字段名统一、缺失字段补默认值 |
| 降级策略 | 每个智能体内置默认数据，AI失败返回预设内容 |
| 知识图谱 | Neo4j图数据库 + 意图识别 + AI生成Cypher |
| 接口暴露 | FastAPI路由，每层都有异常处理和容错 |
